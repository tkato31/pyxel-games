# COGS 仕様書

**タイトル：** COGS  
**サブタイトル：** 足りない歯車を、埋めてください。  
**ジャンル：** パズルゲーム  
**プラットフォーム：** Python / Pyxel  
**最終更新：** 2026-06-27

---

## 1. ゲーム概要

### コンセプト（二層構造）

| 層 | 内容 |
|---|------|
| 表の顔 | 歯車の空きを埋めるパズルゲーム。歯の数・回転方向・速度比を考えて正しい歯車を配置する |
| 裏の顔 | プレイヤー自身がいつの間にか歯車のひとつになっている。気づいた後の感情はプレイヤーに委ねる |

### 3段階構造

| フェーズ | プレイヤーの立場 | 体験 |
|---------|---------------|------|
| 序盤 | 機械を修理する人 | シンプルな作業感。自分は外側にいる |
| 中盤 | 機械の中にいる人 | いつの間にか自分のポジションが固定されている |
| 終盤 | 代替が来る | 自分の歯車の空きに別の誰かが埋めにくる。気づきの瞬間 |

## 4. 歯車オブジェクト

### Gear クラス

```python
{
    "x": int,          # 中心X座標
    "y": int,          # 中心Y座標
    "teeth": int,      # 歯の数
    "radius": int,     # 本体の半径（px）
    "angle": float,    # 現在の回転角度（度数法）
    "speed": float,    # 回転速度（度/フレーム） 正=時計回り、負=反時計回り
    "fixed": bool,     # True=配置済み
    "tooth_len": int,  # 歯の長さ = max(3, radius // 3)
}
```

### 描画構造

1. **歯（台形）** — 各歯を2つの三角形（`pyxel.tri`）で描画
2. **本体（円）** — `pyxel.circ`で塗りつぶし
3. **スポーク（4本）** — 中心から歯の先端まで貫通する線（`pyxel.line`）
4. **中心穴** — `pyxel.circ`（背景色）＋ `pyxel.circb`（リング）＋ 中心点

### 歯の形状パラメータ

```
base_half = pi / teeth * 0.7   （歯の根元の半幅）
tip_half  = pi / teeth * 0.4   （歯の先端の半幅）
```

## 5. 噛み合い判定

### メッシュ距離

```python
def mesh_dist(r1, r2):
    tl1 = max(3, r1 // 3)
    tl2 = max(3, r2 // 3)
    return r1 + r2 + max(tl1, tl2) + 2
```

### 速度比

```python
speed_b = -speed_a * (teeth_a / teeth_b)
```

- 隣接する歯車は必ず逆方向に回転
- 歯数の比率で速度が決まる

### 位相同期

```python
def calc_mesh_angle(driver, driven_teeth):
    step = 360.0 / driven_teeth
    return driver.angle * (-driver.teeth / driven_teeth) + step / 2
```

### 隣接判定

- 配置済み歯車からの距離が `outer_radius + 60` 以内の最近傍を探索
- 隣が見つからない場合は配置不可

## 6. ゲームフロー

### ステージ構成

| ステージ | 固定歯車 | スロット数 | 手持ち | ダミー | 特徴 |
|---------|---------|-----------|--------|--------|------|
| 1 | 8歯×1 | 1 | 2 | 1 | チュートリアル |
| 2 | 8歯×1 | 2 | 3 | 1 | 直線配置 |
| 3 | 10歯×1 | 2 | 3 | 1 | 上下＋左右の分岐配置 |
| 4 | 8歯×1 | 3 | 4 | 1 | 長い直線チェーン |
| 5 | 10歯×1 | 3 | 4 | 1 | 三方向分岐 |

### 操作フロー

1. 画面下の手持ちエリアから歯車をクリックして選択（黄色にハイライト）
2. 空きスロット（?マーク点滅）をクリックして配置を試みる
3. 歯数が正解 → 配置成功（「カチッ」SE、歯車が回転開始）
4. 歯数が不正解 → 「MISS!」表示（赤文字、ガコンSE）
5. 全スロット配置完了 → クリア演出（加速＋黄色点滅＋クリアSE）
6. 2秒後にENTERで次ステージへ

### クリア演出

- 全歯車の回転速度が徐々に加速（`speed_mult = 1.0 + clear_timer * 0.02`）
- 歯車の色が4フレームごとに黄色と通常色で点滅
- 120フレーム後に「PRESS ENTER FOR NEXT」表示

## 8. Phase 2 詳細設計

### 8.1 高解像度化（企画提案）

**目的：** ドットのガタつきを解消し、よりリアルで美しい歯車を描画する

**アプローチ案：**
- 解像度を 320x240 → 480x360 または 640x480 に拡大
- 歯車の半径も比例して拡大（描画ドット数増加で滑らかに）
- 歯の形状パラメータ（base_half, tip_half）の微調整
- アンチエイリアス的な中間色による輪郭のなめらかさ向上
- スポークの太さ（複数ライン描画）の検討

**注意点：**
- Pyxelの最大画面サイズ制約の確認
- ウィンドウ表示サイズとのバランス
- 歯車サイズ変更時はmesh_dist関数により自動でスロット位置が再計算される

### 8.2 配置成功時の発光演出

- 配置直後の数フレーム、歯車を明るい色（10:黄色）で描画
- フレームカウンタで制御（10〜15フレーム程度）

### 8.3 ゴースト表示（配置プレビュー）

- 手持ち歯車を選択中にスロット上にマウスを近づけると、半透明（点滅）で歯車のシルエットを表示
- 配置前にサイズ感を確認できる

### 8.4 ステージ開始演出

- ステージ読み込み時に「STAGE N」を画面中央に大きく表示（2秒間）
- フェードアウト後にプレイ開始

### 8.5 ステージ間トランジション

- クリア後のENTER → 画面が暗転（黒で塗りつぶし）→ 次ステージがフェードイン

### 8.6 やり直し機能

- 配置済み歯車を右クリックで手持ちに戻す
- 以降の連鎖歯車も同時に外れる

### 8.7 マウス追従

- 選択中の歯車の小さなアイコンをマウスカーソル位置に描画
- 描画は半透明（点滅表現）

### 8.8 BGM

- ch0: ベースリズム（低音の規則的パルス）
- ch1: 機械音（金属的な中音域のループ）
- ch2: アクセント（高音域の短いフレーズ）
- 工場の機械が稼働している雰囲気
