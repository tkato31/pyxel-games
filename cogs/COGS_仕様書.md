## 1. ゲーム概要

### コンセプト

モーターの動力を、歯車をつないで、すべての機械に必要な分だけ届けるパズルゲーム。

### パズルの核心

1. **動力分散** — 分岐すると動力が均等に分かれる
2. **回転方向** — 歯車チェーンで方向が反転。機械は特定の方向を要求
3. **増幅と減衰** — 増幅ギア(+1)で動力を補う。減速ギア(-0.5)
4. **リソース制限** — 歯車の種類と数が限られている

### プレイヤーが考えること

- どこで分岐させるか（動力の配分）
- 増幅ギアをどの経路に入れるか
- 回転方向は合うか（奇偶数の制御）

## 4. ゲームメカニクス

### 4.1 動力伝達

- 歯車は上下左右に隣接する歯車と噛み合う（斜めは不可）
- モーターからBFS探索で連鎖的に動力が伝わる
- 孤立した歯車は回転しない

### 4.2 動力分散（v2の核心）

```
モーター(P4) → ギア(P4) → 分岐点(P4)
                              ├→ ギア(P2)  ← 動力が半分に！
                              └→ ギア(P2)  ← 動力が半分に！
```

- 分岐点から**N方向**に分かれると、各方向に**動力/N**が配分される
- 3方向分岐なら各1/3
- 増幅ギアで+1して動力を補える

### 4.3 増幅の加算式

```
P1 → 増幅ギア → P2（+1）
P0.5 → 増幅ギア → P1.5（+1）
```

- 増幅は**掛け算ではなく足し算**
- 入力値によって増幅後の値が変わる
- 「分岐前に増幅」と「分岐後に増幅」で結果が異なる → **位置が重要**

### 4.4 回転方向

```
モーター(↻) → 通常(↺) → 通常(↻) → 通常(↺) ...
            → 同方向(↻) → 通常(↺) ...
```

- 通常・増幅・減速・大型: 方向**逆転**
- 同方向: 方向**維持**
- 機械は特定の方向を要求（矢印アニメーションで表示）

### 4.5 配置ルール

- HandGearから歯車を選択し、グリッドの空きマスに左クリックで配置
- 配置するとアクションを1消費
- 右クリックで配置済み歯車をHandGearに戻す（アクション非消費）
- 固定ギアは右クリックで外せない

### 4.6 クリア条件

- 全ての機械が稼働状態であること
- 稼働 = 隣接歯車の回転方向が正しく、受信動力が要求値以上

### 4.7 動力表示

- 各歯車の下に動力値を「P数値」で表示
- 機械の下に必要動力値を表示（足りないと赤、足りると緑）

## 5. ステージ設計

### ステージ一覧

| Stage | パズルポイント | 考えること |
|-------|--------------|-----------|
| 1 | 直線接続 | つなぐだけ（チュートリアル） |
| 2 | 動力分散 | P2を分岐して2つの機械にP1ずつ届ける |
| 3 | 増幅 | P1モーターに増幅で+1してP2にする |
| 4 | 分岐+増幅 | 分岐でP1に落ちた先に増幅で回復 |
| 5 | 方向+分岐 | 2機械が異なる方向要求。同方向ギアの位置が鍵 |
| 6 | 総合 | P4モーターを分岐+方向制御で2機械に |

## 9. バージョン履歴

### v1（cogs_v1.py, COGS_仕様書_v1.md/html）

- 迷路型ステージ（壁と通路）
- 回転方向パズル（方向のみ）
- 動力は分散なし（そのまま伝達）
- 増幅は乗算式（×2）
- 12ステージ + ボーナスシステム
