## 1. ゲーム概要

### コンセプト（二層構造）

| 層 | 内容 |
|---|------|
| 表の顔 | 歯車の空きを埋めるパズルゲーム。歯の数・回転方向・速度比を考えて正しい歯車を配置する |
| 裏の顔 | プレイヤー自身がいつの間にか歯車のひとつになっている。気づいた後の感情はプレイヤーに委ねる |

### 3段階構造

| フェーズ | プレイヤーの立場 | 体験 |
|---------|---------------|------|
| 序盤 | 機械を修理する人 | シンプルな作業感。自分は外側にいる |
| 中盤 | 機械の中にいる人 | いつの間にか自分のポジションが固定されている |
| 終盤 | 代替が来る | 自分の歯車の空きに別の誰かが埋めにくる。気づきの瞬間 |

## 4. 歯車オブジェクト

### モジュール式サイズ体系

歯車工学のモジュール概念を採用。`module = 4` として `radius = teeth × 2`。

| サイズ | 歯数 | 半径(px) | 用途例 |
|--------|------|---------|--------|
| XL | 20 | 40 | 大型固定歯車、高難度ステージ |
| L | 16 | 32 | 固定歯車の標準サイズ |
| M | 12 | 24 | 配置歯車の標準サイズ |
| S | 8 | 16 | 小型配置歯車 |
| XS | 6 | 12 | 極小歯車、高難度ステージ |
| ダミー | 10, 14 | 20, 28 | 正解でない歯車（同モジュール） |

```python
def gr(teeth):
    return teeth * 2  # module=4
```

### Gear クラス

```python
{
    "x": int,              # 中心X座標
    "y": int,              # 中心Y座標
    "teeth": int,          # 歯の数
    "radius": int,         # 本体の半径（px）
    "angle": float,        # 現在の回転角度（度数法）
    "speed": float,        # 回転速度（度/フレーム）
    "fixed": bool,         # True=配置済み
    "tooth_len": int,      # 歯の長さ = max(3, radius // 3)
    "gear_hand_index": int # 配置元のHandGearインデックス（戻し用）
}
```

### 描画構造

1. **歯（台形）** — 各歯を2つの三角形（`pyxel.tri`）で描画
2. **本体（円）** — `pyxel.circ`で塗りつぶし
3. **スポーク（4本）** — 中心から歯の先端まで貫通する線（`pyxel.line`）
4. **中心穴** — `pyxel.circ`（背景色）＋ `pyxel.circb`（リング）＋ 中心点

### 歯の形状パラメータ

```
base_half = pi / teeth * 0.7   （歯の根元の半幅）
tip_half  = pi / teeth * 0.4   （歯の先端の半幅）
```

## 5. 噛み合い判定

### メッシュ距離

```python
def mesh_dist(r1, r2):
    tl1 = max(3, r1 // 3)
    tl2 = max(3, r2 // 3)
    return r1 + r2 + max(tl1, tl2) + 2
```

### 速度比

```python
speed_b = -speed_a * (teeth_a / teeth_b)
```

- 隣接する歯車は必ず逆方向に回転
- 歯数の比率で速度が決まる

### 位相同期

```python
def calc_mesh_angle(driver, driven_teeth):
    step = 360.0 / driven_teeth
    return driver.angle * (-driver.teeth / driven_teeth) + step / 2
```

### 隣接判定（2段階）

| 判定 | 範囲 | 用途 |
|------|------|------|
| 広域探索（`find_neighbor`） | `outer_radius + 100` | 接続先歯車の探索、スロットのアクティブ表示 |
| 厳密隣接（`is_adjacent`） | `mesh_dist(r1, r2) + 10` | BAD ORDERの整合性チェック |

### BAD ORDER判定

配置時に`is_adjacent`で物理的に隣接する歯車すべてと速度の整合性をチェック。
速度が一致しない場合は「BAD ORDER!」で配置を拒否し、アクションを1消費する。

## 6. ゲームフロー

### ステージ構成

| Stage | 固定歯車 | スロット | HandGear | ダミー | アクション | 特徴 |
|-------|---------|---------|--------|--------|-----------|------|
| 1 | L(16歯)×1 | 1 | 2 | 1 | 2 | チュートリアル |
| 2 | XL(20歯)×1 | 2 | 3 | 1 | 4 | 直線配置、大→小 |
| 3 | L(16歯)×1 | 2 | 3 | 1 | 4 | 上下＋左右の分岐 |
| 4 | XS(6歯)×1 | 3 | 4 | 1 | 5 | 長い直線チェーン、小→大 |
| 5 | L(16歯)×1 | 3 | 4 | 1 | 6 | 三方向分岐 |

### 操作フロー

1. 画面下のHandGearエリアから歯車を左クリックして選択（黄色にハイライト）
2. 空きスロット（?マーク点滅）を左クリックして配置を試みる
3. 歯数が正解かつ回転整合 → 配置成功（「カチッ」SE、歯車が回転開始）
4. 歯数が不正解 → 「MISS!」表示（赤文字、ガコンSE）
5. 回転速度が隣接歯車と不整合 → 「BAD ORDER!」表示
6. 配置済み歯車を右クリック → HandGearに戻す（チェーン歯車も同時に戻る）
7. 全スロット配置完了 → クリア演出
8. アクション数が0で未クリア → GAME OVER（ENTERでリトライ）

### アクション制限

- 配置成功、MISS、BAD ORDERすべてアクションを1消費
- 右クリックでの戻しはアクションを消費しない（回復もしない）
- 残り1以下でACTIONS表示が赤色に
- クリア時もACTIONS数は表示維持

### クリア演出

- 全歯車の回転速度が徐々に加速（`speed_mult = 1.0 + clear_timer * 0.02`）
- 歯車の色が4フレームごとに黄色と通常色で点滅
- 120フレーム後に「PRESS ENTER FOR NEXT」表示

### GAME OVER

- プレイ画面上に赤枠のボックスをオーバーレイ表示
- 「GAME OVER」＋「PRESS ENTER TO RETRY」
- ENTERで同ステージをリトライ（アクション数リセット）

## 10. 修正履歴

### 2026-06-27 Phase 2 実装

#### 歯車配置位置のずれ（Phase 1末期）

**症状：** ステージ4がクリアできない  
**原因：** `try_place`で歯車の位置をメッシュ距離から再計算していたため、スロット位置とずれが生じた。次のスロットが配置済み歯車を隣接として見つけられなくなった。  
**修正：** 歯車をスロット位置にそのまま配置するように変更。スロット位置は`mesh_dist`関数で正しいメッシュ距離に基づいて定義。

#### 回転不整合（配置順バグ）

**症状：** ステージ2で右→中央の順に置くと回転が合わない  
**原因：** 配置順によって速度計算の基準歯車が変わり、チェーン内の速度に矛盾が発生。  
**修正：** BAD ORDER判定を導入。配置時に隣接するすべての歯車と速度の整合性をチェックし、不整合なら配置を拒否。

#### 隣接判定の誤検出

**症状：** ステージ2の右スロットに配置するとBAD ORDERが出る（正しい順序でも）  
**原因：** `find_all_neighbors`の探索範囲（`outer_radius + 60`）が広すぎて、直接噛み合わない2ホップ先の固定歯車も隣接として検出していた。  
**修正：** 2段階の隣接判定を導入。接続先探索は広域範囲（`outer_radius + 100`）、BAD ORDERチェックは`is_adjacent`関数（`mesh_dist + 10`）で厳密判定。

#### GAME OVER判定のタイミング

**症状：** 最後の1アクションでクリアしてもGAME OVERが表示される  
**原因1：** `update`と`draw`でgame_overをclearedより先に判定していた。  
**原因2：** 成功配置後に`check_game_over`を呼んでいなかったケースがあった。  
**修正：** cleared判定をgame_overより常に優先。成功配置でクリアした場合は`game_over = False`を明示的に設定。

#### GAME OVER画面の暗転問題

**症状：** GAME OVER時に画面が暗転してメッセージだけ表示される  
**修正：** プレイ画面の描画を`draw_play_screen`メソッドに切り出し、GAME OVER画面はプレイ画面の上にオーバーレイ表示するように変更。

#### MISS/CLEARメッセージの重なり

**症状：** MISSメッセージが表示中にクリアするとテキストが重なる  
**修正：** クリア判定時に`message`と`message_timer`をリセット。

#### 右クリック戻しのHandGear順序

**症状：** 歯車を戻した時にHANDエリアで歯車の順番が入れ替わる  
**修正：** `HandGear`に`hand_index`を追加。配置時にGearオブジェクトに`gear_hand_index`として記録。戻し時に元のインデックスで復帰し、`hand_index`でソート。
