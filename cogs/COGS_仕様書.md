# COGS 仕様書

**タイトル：** COGS  
**サブタイトル：** 足りない歯車を、埋めてください。  
**ジャンル：** 動力伝達パズルゲーム  
**プラットフォーム：** Python / Pyxel  
**解像度：** 512 x 384  
**最終更新：** 2026-06-28

---

## 1. ゲーム概要

### コンセプト

グリッド（マス目）上に歯車を配置し、モーター（動力源）から機械（目的地）まで動力を伝えるパズルゲーム。

```
[モーター A] → 歯車を配置 → [機械 B を動かす]
```

限られた歯車を使い、障害物を迂回し、**回転方向を考慮して**複数の機械を同時に動かすルートを組み立てる。

### パズルの核心：回転方向

- 歯車チェーンでは、隣接するたびに回転方向が反転する
- 機械は**特定の回転方向**でないと稼働しない
- チェーン内の歯車の数（奇数/偶数）で到達する回転方向が変わる
- 同方向ギアや大型ギアを使って回転方向を制御するのがパズルの鍵

### 二層構造（プロトタイプから継承）

| 層 | 内容 |
|---|------|
| 表の顔 | 動力伝達パズル。歯車をつないで機械を動かす達成感 |
| 裏の顔 | プレイヤー自身がいつの間にか歯車のひとつになっている |

## 2. 画面レイアウト

### 画面構成

| エリア | 位置 | サイズ | 内容 |
|--------|------|--------|------|
| ヘッダー | 上部 | 512x28 | ステージ名、ACTIONS数、GEARS数 |
| グリッド | 中央 | 448x280 (16x10マス) | パズル盤面。マス目に歯車を配置 |
| HandGearエリア | 下部 | 512x84 | 配置可能な歯車の一覧（種類別表示） |

### グリッドマス

- 1マス = 28x28 ピクセル
- 16列 x 10行 = 160マス
- グリッド左上座標: `GRID_X = (512 - 28*16) // 2`, `GRID_Y = 28`
- 歯車のサイズ: `GEAR_RADIUS = 10`, `GEAR_TEETH = 8`
- 迷路型: 壁（配置不可マス）と通路（配置可能マス）で構成

## 3. 登場キャラクター（ギア）

### 3.1 通常ギア（Normal Gear）

| 項目 | 内容 |
|------|------|
| タイプ | `GEAR_NORMAL` |
| サイズ | 1マス |
| 回転伝達 | 逆転（隣接歯車と逆方向に回転） |
| 色 | 灰色 (13) / 選択時 黄色 (10) |
| 用途 | 基本の歯車。動力を伝達し、方向を反転させる |

### 3.2 同方向ギア（Same-Direction Gear）

| 項目 | 内容 |
|------|------|
| タイプ | `GEAR_SAME` |
| サイズ | 1マス |
| 回転伝達 | 同方向（隣接歯車と同じ方向に回転） |
| 色 | シアン (12) |
| HandGear表示 | `=8T` のように `=` マーク付き |
| 用途 | 回転方向を維持する。奇偶数パリティを調整するキーアイテム |

### 3.3 大型ギア（Large Gear）

| 項目 | 内容 |
|------|------|
| タイプ | `GEAR_LARGE` |
| サイズ | 2x2マス（4セル占有） |
| 回転伝達 | 逆転 |
| 色 | 水色 (6) |
| 半径 | `GEAR_RADIUS + 8` |
| HandGear表示 | `L8T` のように `L` マーク付き |
| 用途 | 1手で通常より遠くに動力を届ける。辺が多いため分岐の起点になりやすい |
| 特性 | 通常ギア2個分の距離を1つでカバーし、チェーンのパリティを変える |

### 3.4 固定ギア（Fixed Gear）

| 項目 | 内容 |
|------|------|
| タイプ | `GEAR_FIXED` |
| サイズ | 1マス |
| 回転伝達 | 逆転 |
| 色 | 暗い青 (5) |
| 用途 | 最初から盤面に配置済み。右クリックで外せない。ルートの制約かつ手がかり |

### 3.5 モーター（Motor）

| 項目 | 内容 |
|------|------|
| サイズ | 1マス |
| 色 | 黄色 (10) |
| 表示 | 歯車描画 + 下部に「M」テキスト |
| 用途 | 動力源。常に回転し、隣接歯車に動力を供給。配置変更不可 |

### 3.6 機械（Machine / Target）

| 項目 | 内容 |
|------|------|
| サイズ | 1マス |
| 色 | 未稼働=赤 (8) / 稼働=緑 (11) |
| 表示 | 二重枠 + 回転する矢印アニメーション（必要な回転方向を示す） |
| 用途 | 動力を届ける対象。**指定された回転方向**で隣接歯車が回転していないと稼働しない |
| 方向 | `dir: 1`（時計回り ↻） or `dir: -1`（反時計回り ↺） |

## 4. ゲームメカニクス

### 4.1 動力伝達ルール

- 歯車は上下左右に隣接する歯車と噛み合う（斜めは不可）
- 通常ギア・固定ギア・大型ギア: 噛み合った歯車は**逆方向**に回転
- 同方向ギア: 噛み合った歯車と**同じ方向**に回転
- モーターから連鎖的に動力が伝わる（BFS探索）
- 孤立した歯車（モーターからの連鎖が届かない）は回転しない

### 4.2 回転方向パズル

```
モーター(↻ speed=1.0)
  → 通常ギア1(↺) → 通常ギア2(↻) → 通常ギア3(↺) ...
  → 同方向ギア(↻) → 通常ギア(↺) ...
```

- 通常ギアN個のチェーン: N奇数→逆方向、N偶数→同方向
- 同方向ギアを挟むと**パリティが変わらない**（方向を維持）
- 大型ギアは2マス分の距離を1つでカバーし、**パリティを変える**

### 4.3 配置ルール

- HandGearから歯車を選択し、グリッドの空きマスに配置
- 大型ギアは2x2の空きが必要
- 壁、モーター、機械、固定ギアのマスには配置不可
- 配置するとアクションを1消費
- 右クリックで配置済み歯車をHandGearに戻す（アクション非消費）
- 固定ギアは右クリックで外せない

### 4.4 クリア条件

- 全ての機械（Target）が稼働状態であること
- 稼働 = 隣接マスの歯車が**機械が要求する回転方向**で回転していること

### 4.5 大型ギアの隣接判定

大型ギアは2x2マスを占有するため、通常より多くの隣接セルを持つ:

```
    [隣][隣]
[隣][LG][LG][隣]
[隣][LG][LG][隣]
    [隣][隣]
```

2x2ブロックの外周8セルが隣接セルとなる（内部の4セルは除外）。

## 5. ステージ設計

### ステージ一覧

| Stage | 新要素 | 機械数 | パズルポイント |
|-------|--------|--------|--------------|
| 1 | 通常ギア | 1 | つなぐだけ（チュートリアル） |
| 2 | 固定ギア | 1 | 固定ギアを活かして正しいパリティのチェーンを作る |
| 3 | 同方向ギア | 2 | 2機械の異なる方向要求を同方向ギアで解決 |
| 4 | 大型ギア | 1 | 6セル距離を大型ギアで5チェーンに短縮しパリティ変更 |
| 5 | 固定+同方向 | 2 | 固定ギアを活かしつつ同方向ギアで方向調整 |
| 6 | 全種類 | 2 | 大型+固定+同方向を駆使して2機械を稼働 |

### ステージデータ構造

```python
{
    "name": "STAGE 1",
    "actions": 5,
    "motors": [{"col": 2, "row": 4, "speed": 1.0}],
    "machines": [{"col": 6, "row": 4, "dir": -1}],
    "fixed_gears": [],
    "walls": maze_walls({...}),  # 通路セットから壁を自動生成
    "hand": [
        {"type": GEAR_NORMAL, "teeth": 8, "count": 5},
        {"type": GEAR_SAME, "teeth": 8, "count": 2},
        {"type": GEAR_LARGE, "teeth": 8, "count": 1},
    ],
}
```

### `maze_walls` ヘルパー関数

通路（配置可能セル）のセットを渡すと、それ以外のセルを壁として返す。迷路型ステージの定義を簡潔にする。

```python
def maze_walls(open_cells, cols=GRID_COLS, rows=GRID_ROWS):
    walls = []
    for r in range(rows):
        for c in range(cols):
            if (c, r) not in open_cells:
                walls.append({"col": c, "row": r})
    return walls
```

## 6. UI設計

### ヘッダー

```
COGS    STAGE 1                    ACTIONS: 5  GEARS: 5
```

- ACTIONS: 残りアクション数（1以下で赤色）
- GEARS: 残り配置可能歯車の合計数

### 操作ガイド

```
LEFT CLICK: PLACE / RIGHT CLICK: REMOVE
```

### HandGearエリア

- 歯車の種類ごとに色分け表示
- 通常（灰色）、同方向（シアン `=`）、大型（水色 `L`）
- 歯数と残数を表示
- 選択中は黄色ハイライト

### ゴースト表示

- HandGear選択中にグリッドマスにマウスを合わせると配置プレビュー
- 配置可能マスのみ表示

### 機械の回転方向表示

- 回転する矢印アニメーション（`pyxel.frame_count`で角度を計算）
- 矢印の先端に三角形を描画
- 方向が一目でわかる
