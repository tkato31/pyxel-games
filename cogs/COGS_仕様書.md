# COGS 仕様書

**タイトル：** COGS  
**サブタイトル：** 足りない歯車を、埋めてください。  
**ジャンル：** 動力伝達パズルゲーム  
**プラットフォーム：** Python / Pyxel  
**解像度：** 512 x 384  
**最終更新：** 2026-06-27

---

## 1. ゲーム概要

### コンセプト

グリッド（マス目）上に歯車を配置し、モーター（動力源）から機械（目的地）まで動力を伝えるパズルゲーム。

```
[モーター A] → 歯車を配置 → [機械 B を動かす]
```

限られた歯車を使い、障害物を迂回し、複数の機械を同時に動かすルートを組み立てる。

### 二層構造（プロトタイプから継承）

| 層 | 内容 |
|---|------|
| 表の顔 | 動力伝達パズル。歯車をつないで機械を動かす達成感 |
| 裏の顔 | プレイヤー自身がいつの間にか歯車のひとつになっている |

```
+--------------------------------------------------+

M = モーター（動力源）

```

### 画面構成

| エリア | 位置 | サイズ | 内容 |
|--------|------|--------|------|
| ヘッダー | 上部 | 512x24 | ステージ名、残り歯車数、アクション数 |
| グリッド | 中央 | 448x280 (16x10マス) | パズル盤面。マス目に歯車を配置 |
| HandGearエリア | 下部 | 512x80 | 配置可能な歯車の一覧 |

### グリッドマス

- 1マス = 28x28 ピクセル
- 16列 x 10行 = 160マス
- マス内に歯車1つが入る
- 歯車のサイズは1マスに収まるよう統一（radius=12, teeth*で歯数のみ変化）

## 3. 登場キャラクター（ギア）

### 3.1 通常ギア（Normal Gear）

プレイヤーが配置する歯車。隣接する歯車と噛み合い、動力を伝達する。

```
歯数: 8（標準）
色: 白 (7) / 配置後は各色

  ####
 #    #
#  --  #
# |  | #
#  --  #
 #    #
  ####

ピクセル表現 (12x12):
    ########
   ##      ##
  ##   --   ##
 ##   |  |   ##
##    |  |    ##
##     --     ##
##    |  |    ##
 ##   |  |   ##
  ##   --   ##
   ##      ##
    ########

※ 歯は台形で描画（プロトタイプのGear.draw()を流用）
※ スポーク4本が中心から歯先まで貫通
※ 中心に穴（リング描画）
```

### 3.2 中継ギア（Relay Gear）

小型の歯車。通常ギアより小さく、回転方向を反転させる中継として使う。
グリッド上では1マスを占有するが、描画は小さめ。

```
歯数: 6（小型）
色: シアン (12)
特徴: 通常ギアより小さい描画

   ####
  #    #
 # |  | #
 #  --  #
 # |  | #
  #    #
   ####

※ radius が通常より小さい
※ スポークは2本（十字ではなく縦線のみ）
※ 配置時に水色で点滅して中継であることを示す
```

### 3.3 固定ギア（Fixed Gear）

最初から盤面に配置されている歯車。プレイヤーは移動・削除できない。
ルート設計の制約として機能する。

```
色: 灰色 (13)
特徴: 暗い色、動かせないことを示すロックマーク

  ########
 ##  XX  ##
##  X  X  ##
## X    X ##
##  X  X  ##
 ##  XX  ##
  ########

※ 中心にXマーク（ロック表現）
※ 回転はするが、配置変更不可
※ ルートに組み込む前提で配置されている
```

### 3.4 モーター（Motor）

動力源。ここから回転が始まる。プレイヤーは配置変更不可。

```
色: 黄色 (10)
アイコン: 稲妻マーク付き歯車

  ########
 ## /\ /\ ##
##  \/  \/ ##
## /\ /\ ##
##  \/  \/ ##
 ## /\ /\ ##
  ########
    ⚡

※ 常に回転している
※ 黄色で発光表現（4フレーム点滅）
※ 稲妻マークは中心部に描画
```

### 3.5 機械（Machine / Target）

動力を届ける対象。隣接する歯車から動力を受け取ると稼働する。

```
色: 未稼働=赤 (8) / 稼働=緑 (11)

未稼働:
  +------+
  | STOP |
  | ■■■■ |
  +------+

稼働:
  +------+
  |  GO  |
  | ▶▶▶▶ |
  +------+

※ 1マスを占有
※ 隣接する歯車が回転していれば稼働状態に
※ 稼働すると色が赤→緑に変化し、アニメーション開始
```

### 3.6 お邪魔キャラ（Saboteur）

一定間隔で盤面上を移動し、歯車を外してHandGearに戻す妨害キャラ。

```
色: ピンク (14)
外見: 小さなレンチを持ったキャラ

  ##
 ####
  ##
 #  #
  ##
 #  #
#    #

移動パターン:
  - 盤面上を一定間隔（60フレーム）で1マス移動
  - 配置済みの歯車に到達すると、その歯車をHandGearに戻す
  - モーター・固定ギア・機械には干渉しない
  - 壁は通過できない

※ 移動方向は直線（ステージデータで定義）
※ 到達後は元の位置に戻る（往復）
```

## 4. ゲームメカニクス

### 4.1 動力伝達ルール

- 歯車は上下左右に隣接する歯車と噛み合う（斜めは不可）
- 噛み合った歯車は逆方向に回転する
- モーターから連鎖的に動力が伝わる
- 全ての機械が稼働状態になればクリア

### 4.2 配置ルール

- HandGearから歯車を選択し、グリッドの空きマスに配置
- 壁、モーター、機械、固定ギアのマスには配置不可
- 配置するとアクションを1消費
- 右クリックで配置済み歯車をHandGearに戻す（アクション非消費）

### 4.3 噛み合い判定

プロトタイプの判定ロジックを継承：

```python
speed_b = -speed_a * (teeth_a / teeth_b)
```

グリッドベースなので距離判定は不要。上下左右の隣接マスに歯車があれば噛み合う。

### 4.4 クリア条件

- 全ての機械（Target）が稼働状態であること
- 稼働 = 隣接マスに回転中の歯車が存在すること

### 4.5 動力チェーン

モーターから辿って：
1. モーターに隣接する歯車が回転開始
2. その歯車に隣接する歯車が逆方向に回転
3. 連鎖的に伝播
4. 機械に隣接する歯車が回転していれば機械が稼働

孤立した歯車（モーターからの連鎖が届かない）は回転しない。

## 5. 難易度設計

| レベル | 要素 | 説明 |
|--------|------|------|
| 初級 (Stage 1-3) | モーター1→機械1 | 直線・L字でつなぐ。歯車は余裕あり |
| 中級 (Stage 4-6) | 壁・障害物あり | 迂回ルートの設計が必要 |
| 上級 (Stage 7-9) | 機械が複数 | 分岐ルートで同時に動力を伝える |
| 高難度 (Stage 10-12) | 歯車不足＋固定ギア | 最短ルートを見つける最適化パズル |
| 最高難度 (Stage 13-15) | お邪魔キャラ登場 | 妨害を避けるタイミングと配置の戦略 |
| 最終 (Stage 16) | 立場の逆転 | プレイヤーが歯車になる演出 |

## 7. サウンド設計

プロトタイプのサウンドを継承・拡張。

### SE（ch3）

| 用途 | 継承元 |
|------|--------|
| 配置成功 | sounds[0] |
| ミス / 配置不可 | sounds[1] |
| ステージクリア | sounds[2] |
| 機械稼働 | 新規（機械音） |
| お邪魔キャラ接近 | 新規（警告音） |
| お邪魔キャラが歯車を外す | 新規（ガシャン） |

### BGM

| Music | 用途 | 継承元 |
|-------|------|--------|
| musics[0] | 通常ステージ | プロトタイプ musics[0] |
| musics[1] | タイトル画面 | プロトタイプ musics[1] |
| musics[2] | 警告テンポアップ | プロトタイプ musics[2] |
| musics[3] | 全クリア | プロトタイプ musics[3] |
| musics[4] | お邪魔キャラ出現ステージ | 新規（緊迫感） |

## 9. UI設計

### ヘッダー

```
COGS    STAGE 1                    GEARS: 5  ACTIONS: 8
```

- GEARS: 残り配置可能な歯車数
- ACTIONS: 残りアクション数（1以下で赤色）

### グリッド

- マス目の境界線を薄く描画（色1）
- 空きマスはホバーで明るくハイライト
- 配置可能マスと不可マスの色分け

### HandGearエリア

```
HandGear: [通常x5] [中継x2]
```

- 歯車の種類ごとにグループ表示
- 残数を表示
- 選択中は黄色ハイライト

### ゴースト表示

- HandGear選択中にグリッドマスにマウスを合わせると配置プレビュー
- 配置不可マスでは赤色ゴースト
