1. ゲーム概要

モーターの動力を、歯車をつないで、すべての機械に必要な分だけ届けるパズルゲーム。

パズルの核心
  1. 動力分散 — 分岐すると動力が均等に分かれる
  2. 回転方向 — 歯車チェーンで方向が反転。機械は特定の方向を要求
  3. 増幅（加算式） — 増幅ギア(+1)で動力を補う。位置によって結果が変わる
  4. リソース制限 — 歯車の種類と数が限られている
モーター(P4) → ギア → 分岐点
                       ├→ ギア(P2) → 増幅(P3) → 機械A ✓
                       └→ ギア(P2) → 機械B ✓

4. ゲームメカニクス

4.1 動力分散（v2の核心）

モーター(P4) → ギア(P4) → 分岐点(P4)
                              ├→ ギア(P2)  ← 動力が半分に！
                              └→ ギア(P2)  ← 動力が半分に！

分岐点からN方向に分かれると、各方向に動力/Nが配分される。

4.2 増幅の加算式

入力増幅後計算
P1.0P2.01.0 + 1.0
P0.5P1.50.5 + 1.0
P2.0P3.02.0 + 1.0

分岐前に増幅と分岐後に増幅で結果が異なる → 位置が重要！

分岐前: P1 → Amp(P2) → 分岐 → 各P1
分岐後: P1 → 分岐 → 各P0.5 → Amp(P1.5)

4.3 回転方向

モーター(↻) → 通常(↺) → 通常(↻) → 通常(↺) ...
            → 同方向(↻) → 通常(↺) ...
ギアタイプ方向動力
通常 / 増幅 / 減速 / 大型 / 固定逆転各種
同方向維持そのまま

4.4 クリア条件

  • 全ての機械が稼働状態であること
  • 稼働 = 隣接歯車の回転方向が正しく、受信動力が要求値以上
  • デュアル機械 = 2つ以上の隣接歯車が回転 + 動力が要求値以上

5. ステージ設計

Stageパズルポイントプレイヤーが考えること
1直線接続つなぐだけ（チュートリアル）
2動力分散P2を分岐して2つの機械にP1ずつ届ける
3増幅P1モーターに増幅で+1してP2にする
4分岐+増幅分岐でP1に落ちた先に増幅で回復
5方向+分岐2機械が異なる方向要求。同方向ギアの位置が鍵
6総合P4モーターを分岐+方向制御で2機械に

9. バージョン履歴

v1（cogs_v1.py）

  • 迷路型ステージ（壁と通路）
  • 回転方向パズル（方向のみ）
  • 動力は分散なし（そのまま伝達）
  • 増幅は乗算式（×2）
  • 12ステージ + ボーナスシステム
  • 課題: 「並べるだけ」でパズル要素が弱い

v2（cogs.py）← 現在

  • 自由配置グリッド（10x10、壁なし）
  • 動力分散メカニクス（分岐で動力が分かれる）
  • 増幅は加算式（+1）→ 位置が重要
  • 6ステージ（動力分散の基本パターン）
  • タイトル画面リニューアル
  • 課題: ステージ設計の深化、チュートリアル整備
