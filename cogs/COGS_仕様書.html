1. ゲーム概要

モーターの動力を、歯車をつないで、すべての機械に必要な分だけ届けるパズルゲーム。

パズルの核心
  1. 動力分散 — 分岐すると動力が均等に分かれる
  2. 回転方向 — 歯車チェーンで方向が反転。機械は特定の方向を要求
  3. 増幅（加算式） — 増幅ギア(+1)で動力を補う。位置によって結果が変わる
  4. リソース制限 — 歯車の種類と数が限られている
モーター(P4) → ギア → 分岐点
                       ├→ ギア(P2) → 増幅(P3) → 機械A ✓
                       └→ ギア(P2) → 機械B ✓

4. ゲームメカニクス

4.1 動力分散（v2の核心）

モーター(P4) → ギア(P4) → 分岐点(P4)
                              ├→ ギア(P2)  ← 動力が半分に！
                              └→ ギア(P2)  ← 動力が半分に！

分岐点からN方向に分かれると、各方向に動力/Nが配分される。

4.2 増幅の加算式

入力増幅後計算
P1.0P2.01.0 + 1.0
P0.5P1.50.5 + 1.0
P2.0P3.02.0 + 1.0

分岐前に増幅と分岐後に増幅で結果が異なる → 位置が重要！

分岐前: P1 → Amp(P2) → 分岐 → 各P1
分岐後: P1 → 分岐 → 各P0.5 → Amp(P1.5)

4.3 回転方向

モーター(↻) → 通常(↺) → 通常(↻) → 通常(↺) ...
            → 同方向(↻) → 通常(↺) ...
ギアタイプ方向動力
通常 / 増幅 / 減速 / 大型 / 固定逆転各種
同方向維持そのまま

4.4 クリア条件

  • 全ての機械が稼働状態であること
  • 稼働 = 隣接歯車の回転方向が正しく、受信動力が要求値以上
  • デュアル機械 = 2つ以上の隣接歯車が回転 + 動力が要求値以上

7. ヘルプ画面（4ページ）

タイトル画面で「H」キーを押すと表示。クリックまたは→キーで次ページ、←キーで前ページ。

ページ1: 遊び方

遊び方

モーターから機械まで歯車をつないで動力を届けよう。
十分な動力を正しい回転方向で届けると機械が稼働する。

操作内容
左クリック歯車を選択 / 配置
右クリック配置した歯車を外す
Q終了

目標: すべての機械を緑色にすること。
赤 = 動力不足 または 回転方向が違う。

ページ2: ギアの種類

通常ギア
逆転 / P同じ
同方向ギア =

増幅ギア +
逆転 / P+1.0
減速ギア -

大型ギア L
2x2 / 逆転
ギア回転方向動力用途
通常ギア逆転そのまま基本の伝達
同方向ギア =維持そのまま方向の調整に使う
増幅ギア +逆転+1.0分岐で失った動力を補う
減速ギア -逆転-0.5動力を下げる
大型ギア L逆転そのまま2x2マス。分岐の起点に有利

ページ3: 動力の分岐

歯車が2つ以上の歯車に接続すると、動力が均等に分かれる！

増幅ギアの位置が重要！

パターン計算各経路の動力
分岐前に増幅P1 → 増幅(P2) → 分岐各 P1.0
分岐後に増幅P1 → 分岐 → P0.5 → 増幅増幅側 P1.5

同じ増幅でも、置く場所で結果が変わる。

ページ4: モーターと機械

モーター（黄色）
動力源。動力値（P1, P2...）を出力する。常に回転している。

機械（赤 / 緑）
動力を届ける対象。矢印が必要な回転方向を示す。
数字が必要な動力値を示す。条件を満たすと赤→緑に変化。

デュアル機械（三重枠）
2つ以上の隣接歯車が回転している必要がある。
すべての隣接歯車の動力が合算される。

10. バージョン履歴

v1（cogs_v1.py）

  • 迷路型ステージ（壁と通路）
  • 回転方向パズル（方向のみ）
  • 動力は分散なし（そのまま伝達）
  • 増幅は乗算式（×2）
  • 12ステージ + ボーナスシステム
  • 課題: 「並べるだけ」でパズル要素が弱い

v2（cogs.py）← 現在

  • 自由配置グリッド（10x10、壁なし）
  • 動力分散メカニクス（分岐で動力が分かれる）
  • 増幅は加算式（+1）→ 位置が重要
  • 6ステージ（動力分散の基本パターン）
  • タイトル画面リニューアル
  • 課題: ステージ設計の深化、チュートリアル整備
